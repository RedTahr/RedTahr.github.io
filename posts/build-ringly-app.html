Building the Ringly mobile apps

I bought my wife a Ringly ring a year or two before the company closed its doors. So it was a bit disappointing when she came to install the Ringly app on a replacement phone, and the app was no longer available.

I had a look to see if Ringly had posted any code online and came up empty handed.

A while past and someone replied to a social media post of mine about trying to hunt down the app. I mentioned having seen some of the devs listed on a work related social media site. Within a month Ringly had posted Android and iOS code repositories for their apps; I'm pretty sure they hadn't been there before. So thank you to that lady for sparking off that action.

The easy one was Android

I've done some light native Android work. The first hurdle was deciding which version of Android Studio to work with. After that, I removed the Firmware (DFU) updating logic, and the login page, and the app was happy to track my steps and blink at me for notifications.

The less easy one, iOS.

During my attempts at this there's been the occasional social media post about it. And I've had a few replies and DMs enquiring if I had had success. I'm not a native iOS dev, so attempts at this have been, at best, annual.

One kind soul even shared their backed up IPA. So I edited the info.plist for it, to associate it with our account, but failed to work out how to get it to run on a device under our apple account.

Recently someone again asked if I'd had success. Which sparked off this most recent attempt.

At this point my highlights for getting close to this building include:

  • use XCode 8.x
  • you may find yourself manually downloading the built framework files from github if Carthage is unhappy
  • generally use software from before 2019 when Ringly posted their code
  • ditch crashalytics, firmware updating (DFU), Fabric, Airship, login logic, push notification requirement, for starters

The journey continues....
